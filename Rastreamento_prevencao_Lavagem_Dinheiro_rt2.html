01

O problema que o projeto resolve

A lavagem de dinheiro é um crime financeiro que disfarça a origem ilícita de fundos, tornando-os aparentemente legítimos. Ela afeta a estabilidade econômica e a integridade dos sistemas financeiros globalmente.

Contexto fictício: os dados, clientes e transações utilizados neste projeto são completamente fictícios e têm finalidade exclusivamente demonstrativa. Nenhuma informação real foi utilizada.

🔴

Colocação

Inserção dos recursos ilícitos no sistema econômico via depósitos, compra de instrumentos ou bens, frequentemente em países com menor regulação.

🟠

Ocultação

Quebra da trilha contábil por meio de transferências eletrônicas para contas anônimas, dificultando rastreamento por autoridades.

🟢

Integração

Os recursos retornam ao sistema econômico formal, aparentemente "limpos", prontos para uso pelas organizações criminosas.

A análise de dados oferece uma abordagem baseada em evidências para identificar e mitigar essas atividades suspeitas, tornando-se um aliado essencial do Compliance.

02

Pipeline ETL com Integration Service

O coração operacional do projeto: a extração, transformação e carga dos dados de movimentações financeiras que alimentam toda a análise.

⚙️

O que é ETL?

Processo de Extração, Transformação e Carga. Coleta dados de diversas fontes, os transforma em formato adequado e os carrega num destino estruturado como um Data Warehouse.

🔗

Integration Service

Componente de plataformas como Microsoft SSIS ou Informatica PowerCenter. Gerencia o fluxo de dados entre fontes e destinos com interface gráfica intuitiva.

Fonte dos dados: um arquivo .csv com informações de clientes e operações realizadas em D-1 (dia anterior ao processamento). Esse ciclo garante que as análises reflitam sempre as movimentações mais recentes disponíveis.

Fluxo das etapas do pacote DTSX

1

Limpeza da tabela temporária

Antes de iniciar a carga, a tabela de staging é limpa para garantir que não haja duplicidade ou dados residuais de execuções anteriores.

2

Leitura do arquivo origem

O arquivo .csv é lido, validado e seus registros são carregados na tabela destino do banco de dados, incluindo dados de clientes e operações do dia anterior.

3

Execução da Procedure de atualização

Uma procedure SQL é disparada para consolidar e atualizar as tabelas relacionadas a clientes e operações com os dados recém-carregados.

4

Tabelas disponíveis para análise

As tabelas de clientes e operações estão atualizadas e prontas para alimentar as regras de detecção e os relatórios analíticos.

03

Execução de Regras de Detecção

Com os dados carregados, o motor de regras entra em ação. A primeira regra avalia movimentações de valores acima de um limiar definido pelo Compliance.

🚨

Regra: Movimentações de Altos Valores

Identifica transações com valor igual ou superior ao Parâmetro01 — definido pela área de Compliance. Todas as movimentações no período analisado são verificadas automaticamente.

📂

Segregação dos resultados

Após a análise, as movimentações que acionam a regra são separadas em tabelas dedicadas, criando uma fila de trabalho para verificação pelas áreas responsáveis.

Resultado desta etapa: as operações suspeitas ficam isoladas em tabelas independentes, prontas para revisão pelo time de Compliance — sem misturar dados limpos com alertas.

04

Consultas e Análises com SQL + Python

A análise aprofundada combina a precisão do SQL com a expressividade do Python para transformar dados brutos em insights acionáveis.

Conexão com SQL Server via Python

Usando a biblioteca pymssql, Python se conecta diretamente ao banco para executar queries e retornar DataFrames prontos para análise.

Python · SQL Server Connection 
# Importando bibliotecas essenciais
import pandas as pd
import pymssql as sql

# Criando conexão com o banco
conexao = sql.connect('servidor', 'usuario', 'senha', 'banco_de_dados')

# Consultando alertas gerados
df = pd.read_sql_query("""
    SELECT COUNT(*) AS quantidade 
    FROM dbo.tgr_alertas
    WHERE cd_enquadramento = 1 
      AND dt_movimentacao = '2020-07-31'
""", conexao)

conexao.close()
